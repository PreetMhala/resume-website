Custom Diffusion Noise Schedule (EDM2, CelebA)

Stony Brook University • Prof. Dimitris Samaras

Oct 2025 - Dec 2025 Graduate Research Project (CSE 527)

Problem Statement

Diffusion models are highly sensitive to their noise schedules, yet there is no clear consensus on which schedules perform best across datasets and training regimes. This project investigates how different noise schedules affect training stability, convergence speed, and sample quality.

Methodology

Using NVIDIA's EDM2-XS architecture, trained multiple diffusion models on CelebA (64×64) under five noise schedules: Karras-ρ, Linear, Cosine, Logarithmic, and Quadratic. Extended to conditional diffusion with 40 binary facial attributes and facial landmark conditioning.

Results

  • Linear, Logarithmic, and Quadratic schedules consistently outperformed Karras-ρ and Cosine
  • Achieved best FID of 10.75 on full CelebA test set
  • Attribute-conditioned FID ≈ 20.26, landmark-conditioned FID ≈ 28.39
  • Smoother noise progressions improve convergence speed

High-Performance 2PC-Sharded KV-Store

Stony Brook University

Nov 2025 - Dec 2025 Distributed Systems

Overview

Developed a high-performance sharded Key-Value store integrating Multi-Paxos for intra-shard replication and Two-Phase Commit (2PC) for atomic cross-shard transactions. Achieves up to 50x throughput gains using optimistic pipelining via asyncio while maintaining linearizability.

Key Innovations

  • Dynamic resharding for load balancing
  • Fine-grained Two-Phase Locking (2PL)
  • Automated state transfer with log compaction
  • 3,000+ concurrent transactions handling capability

Linear PBFT Consensus Engine

Stony Brook University

Oct 2025 - Nov 2025 Distributed Systems

Overview

Developed a Linear PBFT consensus engine optimizing Byzantine Fault Tolerance by reducing message complexity from O(N²) to O(N) through leader-based vote aggregation. Uses State Machine Replication (SMR) and Ed25519 cryptographic signatures.

Key Features

  • Aggregator Pattern for Pre-Prepare, Prepare, and Commit phases
  • Robust View Change protocol with asyncio-based JSON-RPC
  • 100% data integrity across 7-node cluster
  • Tested with Byzantine equivocation, network partitions, and crash failures

Paxos Consensus Ledger - Distributed Banking System

Stony Brook University

Sep 2025 - Oct 2025 Distributed Systems

Overview

Fault-tolerant distributed banking system using Multi-Paxos consensus algorithm ensuring strong consistency across a 5-node cluster. Features dynamic leader election, State Machine Replication, checkpointing, and log gap-filling.

Workflow

  • Client sends transaction to current leader
  • Leader initiates Paxos rounds to achieve quorum
  • Transaction committed and applied to account balances
  • Zero data loss during simulated leader failures and network partitions
